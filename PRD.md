# 公众号文章爬取 + AI 搜索系统 PRD

## 1. 项目概述

### 1.1 产品名称

**PAI** (Personal Article Intelligence) - 个人文章智能管理系统

### 1.2 产品定位

一个能够自动爬取公众号文章、本地存储管理，并通过AI进行智能搜索和总结的工具。

### 1.3 目标用户

- 需要收集整理特定作者文章的个人用户
- 希望通过AI快速检索和总结文章观点的用户

---

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 智能作者发现

| 功能点       | 描述                                           |
| ------------ | ---------------------------------------------- |
| 单文章识别   | 用户提供一篇文章URL，系统自动提取作者信息      |
| 作者档案     | 建立作者档案，包含：名称、公众号ID、头像、简介 |
| 历史文章发现 | 通过搜索引擎找到该作者的所有历史文章           |

#### 2.1.2 文章爬取与存储

| 功能点       | 描述                                     |
| ------------ | ---------------------------------------- |
| 全量爬取     | 首次添加作者时，爬取所有可发现的历史文章 |
| 增量更新     | 后续只爬取新发布的文章                   |
| Markdown存储 | 每篇文章保存为独立的Markdown文件         |
| 元数据管理   | 每篇文章包含YAML Frontmatter元数据       |

#### 2.1.3 AI搜索与总结

| 功能点       | 描述                                                                 |
| ------------ | -------------------------------------------------------------------- |
| 作者筛选     | 可指定搜索特定作者的文章                                             |
| 观点定位     | 根据关键词定位到相关文章                                             |
| AI智能总结   | 接入真实LLM服务（OpenAI/SiliconFlow/Gemini），对搜索结果进行智能总结 |
| 原文引用     | 总结中标注引用来源文章                                               |
| 多提供商支持 | 支持配置不同的AI服务提供商和模型                                     |
| 智能降级     | AI服务不可用时自动回退到基础搜索                                     |

### 4.3 Cline AI 交互流程

**场景1：搜索特定作者的观点**

```
用户：搜索"猫"关于"价值投资"的观点

Cline：
1. 读取 authors.json，确认"猫"存在
2. 读取 wechat/猫/ 目录下的所有文章
3. 筛选包含"价值投资"关键词的文章
4. 阅读相关文章，提取观点
5. 输出：
   - 找到3篇相关文章
   - 文章1《XXX》：观点...
   - 文章2《YYY》：观点...
   - 综合总结：猫认为价值投资的核心是...
```

**场景2：跨作者搜索**

```
用户：所有作者对"AI投资"的看法

Cline：
1. 遍历所有作者目录
2. 搜索相关文章
3. 对比不同作者观点
4. 输出综合对比分析
```

### 5.3 关键算法

#### 作者发现流程

```python
def discover_author(article_url):
    # 1. 下载文章页面
    html = fetch(article_url)

    # 2. 解析作者信息
    author_name = extract_author_name(html)
    biz_id = extract_biz_id(html)

    # 3. 搜索该作者的所有文章
    articles = search_sogou(author_name, biz_id)

    # 4. 保存作者信息
    save_author({
        'name': author_name,
        'biz_id': biz_id,
        'articles': articles
    })
```

#### 增量更新算法

```python
def incremental_update(author_name):
    author = load_author(author_name)
    last_crawl = author['last_crawl']

    # 1. 获取最新文章列表
    new_articles = fetch_new_articles(author, since=last_crawl)

    # 2. 过滤已存在的文章
    existing_ids = load_existing_article_ids(author)
    to_crawl = [a for a in new_articles if a['id'] not in existing_ids]

    # 3. 爬取新文章
    for article in to_crawl:
        crawl_article(article)

    # 4. 更新时间戳
    author['last_crawl'] = now()
    save_author(author)
```

## 6. 非功能需求

### 6.1 性能要求

- 单篇文章爬取时间 < 5秒
- 支持并发爬取（可配置并发数）
- 索引构建时间 < 1秒（1000篇文章）

### 6.2 可靠性

- 网络异常自动重试（3次）
- 断点续传：中断后可从上次继续
- 数据校验：确保文章完整性

### 6.3 可扩展性

- 支持多平台（微信公众号、小红书等）
- 模块化设计，易于添加新平台

### 6.4 安全性

- 遵守robots.txt
- 请求间隔控制（默认1-3秒随机）
- User-Agent轮换

## 7. 风险与应对

| 风险         | 影响             | 应对方案                         |
| ------------ | ---------------- | -------------------------------- |
| 搜狗反爬     | 无法获取文章列表 | 请求间隔、代理池、验证码识别     |
| 微信页面改版 | 解析失败         | 监控页面结构，及时更新解析器     |
| 文章获取不全 | 数据不完整       | 多源搜索（搜狗+微信读书）        |
| 存储空间不足 | 无法保存         | 支持配置存储路径、自动清理旧数据 |
